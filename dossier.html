01 — The problem
Online commerce systematically deceives consumers

The EU Omnibus Directive (2022) and the Digital Services Act (2023) require European e-commerce to eliminate dark patterns. Fines for non-compliance reach 4% of global revenue. Consumer organizations are under pressure to offer digital tools to their members. Kibbo is exactly that tool — already built, already approved, ready to be deployed.

09 — Frequently asked questions
Common buyer questions
Direct answers to the most common questions in the evaluation process
Why are you selling the extension if it works and has value?

My focus has completely changed. I'm currently developing a B2B SaaS project that requires my full dedication. The original plan for Kibbo was to grow through social media with demo videos — a channel that has a learning curve I can't afford right now. That's why I believe this asset is a better fit in the hands of someone who already has an audience, marketing team, or distribution structure. The project is finished and works; what it needs is visibility, not more development.

What was the planned roadmap for future versions?

The plan was divided into three phases. The first was adaptation to the English-speaking market: full interface translation and review of the main stores for the United States and United Kingdom. The second was the development of the mobile version, taking advantage of the fact that current mobile browsers already support Chrome extensions. The third was expansion to LATAM, adapting detection rules to Latin American market stores and patterns. All three phases are designed, documented, and fully executable by the buyer.

Is there a risk that Google will remove the extension from the Chrome Web Store?

The risk is minimal. The latest version has just passed Google's full technical review, which is the strictest process in an extension's lifecycle. The code is completely transparent — no obfuscated scripts, no hidden behavior — and the Privacy Policy details exactly how each piece of data is handled. Kibbo is 100% compliant with current Chrome Web Store policies. The buyer inherits an approved extension with a permanent ID, meaning they can publish new versions without going through an additional identity review.

What are the fixed maintenance costs?

The infrastructure is designed to be practically free. All main analysis happens in the user's browser, so there are no database servers or cloud processing. Vercel, Cloudflare Workers, and Zoho Mail are on permanent free plans. The only fixed annual cost is domain renewal, approximately €12. The only variable cost is the Anthropic AI API, which only activates when the user explicitly uses advanced features — with 100 active daily users the estimated cost is under €15 per month.

How is the technical handover done?

The entire process can be completed in under 48 hours using each platform's official channels. The GitHub repository is transferred directly to the buyer with full development history. The Chrome Web Store extension is transferred by adding the buyer as owner of the developer group, with no need to republish it or lose the review history. The domain is transferred via Namecheap's standard EPP code process. The Cloudflare Worker, corporate email, and Substack account are delivered with credentials and access documentation. I also include a two-hour session to explain the complete architecture and answer any technical questions.

Is a technical team needed to operate the extension?

For day-to-day commercial management, no. The extension runs autonomously and the only moments that require technical intervention are version updates, which happen when a major store changes its design and its selectors need updating. That work can be done by any frontend developer with basic JavaScript experience in a couple of hours, using the technical documentation delivered with the acquisition. If the buyer wants to rebrand the extension with their own identity — logo, colors, name — I can make those changes myself and deliver the customized version before closing.

What user data does the extension collect?

Kibbo does not collect any browsing data. Dark pattern analysis, price history, and all user preferences are stored exclusively in the browser's own local storage (chrome.storage.local). There is no external database, no visited URLs are sent to any server, and no personally identifiable data is recorded. The only calls to external servers are from the five AI features, which only send the public text of the analyzed page — never user data. This greatly simplifies GDPR compliance: since there is no processing of personal data on own servers, registration and notification obligations are minimal.

Is it ready to generate revenue from day one?

Yes. There are three monetization models that can be activated without modifying the extension's core logic. The first is the freemium subscription model: gating advanced features (Certified Snapshot, AI T&C analysis, Zenith Mode) behind a paid plan. The second is affiliate links: the smart price comparison tool is already designed to accept affiliate tags for each recommended store, turning every purchase into recurring revenue. The third is the white-label model for organizations: offering the extension with their own branding to other entities (professional associations, mutual funds, insurers) as a member retention tool.

What support is offered after the acquisition?

The process includes delivery of all written technical documentation: system architecture, feature reference, infrastructure guide, and marketing strategy. If the buyer wants to adapt the extension to their brand, I can apply logo, color, and typography changes in the code before closing. The documentation covers the rest.
